# Documentation

The documentation is split by purpose:

- `decisions/` contains architecture decision records and migration notes.
- `background/` contains design notes and public references that informed the framework.
- `release-checklist.md` lists the package, install, verification, and skill checks required before distribution.

Start with the root `README.md` if you want to run Assay. Read `decisions/ADR-0001-framework-structure.md` if you want to understand why the framework is organized around references, analyses, systems, and iterations.
