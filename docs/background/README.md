# Background

These notes explain the design background behind Assay. They are not required for day-to-day use, but they help readers understand why the framework has a CLI, a managed `.framework/` layer, and a references-to-knowledge workflow.

- `trellis-inspired-design-notes.md` explains the workflow-system patterns that influenced Assay.
- `design-references.md` lists the public references that informed CLI behavior, versioning, update policy, and repository governance.
