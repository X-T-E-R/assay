# Background

These notes explain the design background behind Assay. They are not required for day-to-day use, but they help readers understand why the framework has a CLI, a managed `.assay/` layer, and a references-to-knowledge workflow.

- `design-principles.md` explains the workspace and lifecycle principles behind Assay.
- `design-references.md` lists the public references that informed CLI behavior, versioning, update policy, and repository governance.
